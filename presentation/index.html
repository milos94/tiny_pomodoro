tiny_pomodoro — timer callback


void tick_cb(void*) {
    if (!app.running) return;
    if (app.seconds_left > 0) {
        --app.seconds_left;
        update_timer_label();
        Fl::add_timeout(1.0, tick_cb);   // re-arm every second
        return;
    }
    stop_timer();
    play_wav(SOUND_PATH);
    if (app.current == Session::Focus) {
        ++app.pomodoro_count;
        reset_session(app.pomodoro_count % kLongBreakEvery == 0
                          ? Session::LongBreak : Session::ShortBreak);
    } else {
        reset_session(Session::Focus);
    }
    app.running = true;
    Fl::add_timeout(1.0, tick_cb);       // auto-start next session
}

The Tiny Build

  • Clang + LLD — size-first optimization
  • 10 compiler flags, 5 linker flags
  • Grouped by what they do — not by which tool eats them

But first: where in the build does a flag actually act?

From .cpp to a running process

Where each flag fires

The basics

optimiser
  • -Oz — optimise for size above all else
  • Starts from the -O2 pipeline
  • Sets LLVM's minsize: inlining threshold 225 → 5
  • No loop unrolling, no vectorising

Trade-off: slower hot loops. A pomodoro timer sleeps 99.99% of its life.

The basics

front-end
  • -fno-exceptions — no landing pads, no .gcc_except_table, no personality routine
  • -fno-rtti — no type_info, no dynamic_cast

These act in the front-end: they change what IR is generated at all.

Cost: no library that throws across its API. No dynamic_cast.

what -flto=thin does to the pipeline

Dead code & dead data

  • optimiser-flto=thin — defer codegen to link; whole-program view
  • codegen-ffunction-sections — one ELF section per function
  • codegen-fdata-sections — one ELF section per global
  • linker-Wl,--gc-sections — drop every section nothing reaches
  • linker-Wl,--icf=all — fold byte-identical functions into one
  • codegen-fmerge-all-constants — the same, for .rodata

Three layers: LTO kills dead IR before codegen · GC kills dead sections after it · ICF merges the survivors.
The sections flags exist only so --gc-sections has something to collect.

Deferring codegen is why LTO sees across translation units

Unwind metadata

codegen
  • -fno-asynchronous-unwind-tables — does all the work
  • -fno-unwind-tables — a no-op on x86-64
on top of -Oz .eh_frame
(neither) 920 B
-fno-unwind-tables 920 B — unchanged!
-fno-asynchronous-unwind-tables 696 B
+ -fno-exceptions 72 B

x86-64 mandates async unwind tables, so clang defaults to them — and that default implies -funwind-tables, silently overriding the other flag. .eh_frame only collapses when -fno-exceptions and the async flag fire together.

Symbol surfaces

codegenlinker

Three flags that look interchangeable. They touch three different tables.

Flag Affects Loaded at runtime?
-fvisibility=hidden .dynsym — who may call me yes
-Wl,--as-needed DT_NEEDED — which libs to load yes
-Wl,-s .symtab — names for debuggers never

-fvisibility=hidden is a runtime win (fewer relocations, faster startup). -Wl,-s is a pure disk win — it deletes bytes the kernel never even loads. Zero overlap.

The flags never saw FLTK

  • src/main.cpp — 350 lines — got all 15 flags
  • libfltk.a — 3.3 MB — got none of them

We link a pre-built /usr/local/lib/libfltk.a: opaque machine code, compiled separately with -O3, exceptions on, RTTI on. --gc-sections, --icf and LTO stop at its edge.

Your flags stop at your code.

So build FLTK with the app


add_subdirectory(${FLTK_SOURCE_DIR} fltk_src)   # FLTK is now part of OUR build

foreach(t fltk fltk_images fltk_gl)             # ...so our flags reach it
    target_compile_options(${t} PRIVATE ${TINY_CXX_FLAGS})
endforeach()
pre-built FLTK 850,536 B
FLTK built with us 327,552 B   −61%

Its objects become LLVM IR bitcode instead of ELF relocatable — the optimiser can finally see inside the library.

The Trade-off

  • Build time. FLTK takes minutes. The archive was instant.
  • -fno-exceptions / -fno-rtti on someone else's library. FLTK 1.4 survives it. Not every library will.
  • You now own the library's build.

For most projects, keeping the fast build is the right call.
Just know what you are paying for it.

tiny_pomodoro_pro — shared Timer (timer.hpp)


class Timer {
public:
    enum class State { Stopped, Running, Paused };

    explicit Timer(int seconds = kDefaultFocusMin * 60) noexcept
        : initial_(seconds), remaining_(seconds) {}

    ~Timer() {
        stop_flag_ = true;
        if (worker_.joinable()) worker_.join();
    }

    void start() {
        if (state_ == State::Running) return;
        state_ = State::Running;
        if (!worker_.joinable())
            worker_ = std::thread(&Timer::run, this);
    }
    void pause() noexcept { state_ = State::Paused; }
    void reset() noexcept { remaining_ = initial_; state_ = State::Stopped; }

tiny_pomodoro_pro — the worker thread


private:
    void run() noexcept {
        using namespace std::chrono_literals;
        while (!stop_flag_) {
            if (state_ != State::Running) {
                std::this_thread::sleep_for(100ms);
                continue;
            }
            std::this_thread::sleep_for(1s);
            if (state_ != State::Running) continue;
            if (remaining_ > 0 && --remaining_ == 0)
                state_ = State::Stopped;
        }
    }

    int         initial_, remaining_;
    State       state_{State::Stopped};
    bool        stop_flag_{false};
    std::thread worker_;
};

tiny_pomodoro_pro — centring text by hand


// Xlib has no "label" — you measure the string and place the pixels.
auto draw_centered = [&](XFontStruct* fnt, int bx, int by,
                         unsigned int bw, unsigned int bh,
                         std::string_view text) {
    if (!fnt || text.empty()) return;
    const int len = static_cast<int>(text.size());
    const int tw  = XTextWidth(fnt, text.data(), len);
    const int tx  = bx + (static_cast<int>(bw) - tw) / 2;
    const int ty  = by + static_cast<int>(bh) / 2
                       + (fnt->ascent - fnt->descent) / 2;
    XSetFont(dpy, gc, fnt->fid);
    XDrawString(dpy, win, gc, tx, ty, text.data(), len);
};

tiny_pomodoro_pro_plus — raw terminal


termios g_orig_termios{};

void restore_terminal() noexcept {
    tcsetattr(STDIN_FILENO, TCSANOW, &g_orig_termios);
    std::fputs("\033[?25h\033[0m\n", stdout);   // show cursor again
    std::fflush(stdout);
}

void setup_raw_terminal() {
    tcgetattr(STDIN_FILENO, &g_orig_termios);
    std::atexit(restore_terminal);

    termios raw     = g_orig_termios;
    raw.c_lflag    &= ~static_cast<tcflag_t>(ICANON | ECHO);
    raw.c_cc[VMIN]  = 0;
    raw.c_cc[VTIME] = 0;              // non-blocking read

    tcsetattr(STDIN_FILENO, TCSANOW, &raw);
    std::fputs("\033[?25l", stdout);  // hide cursor
}

tiny_pomodoro_pro_plus — render


void render(const Timer& t, Session sess, int pomodoros,
            int focus_min, int short_min, int long_min) {
    std::fputs("\033[H", stdout);   // home cursor — no clear, no flicker
    std::fputs("\033[1m┌───────────────────────────┐\033[0m\n", stdout);

    const auto name = name_of(sess);
    std::printf("\033[1m│  %-16.*s   #%-3d │\033[0m\n",
                static_cast<int>(name.size()), name.data(), pomodoros);

    const int secs = t.seconds_left();
    std::printf("\033[1m│      \033[32m%02d:%02d\033[0m\033[1m"
                "                │\033[0m\n", secs / 60, secs % 60);

    const char* dot = t.is_running() ? "\033[32m●\033[0m\033[1m"
                                     : "\033[33m■\033[0m\033[1m";
    std::printf("\033[1m│      %s %s          │\033[0m\n",
                dot, t.is_running() ? "Running" : "Paused ");
    std::fputs("\033[1m└───────────────────────────┘\033[0m\n", stdout);
    std::fflush(stdout);
}

tiny_pomodoro_pro_plus — main loop


while (true) {
    char c = 0;
    if (read(STDIN_FILENO, &c, 1) == 1) {
        switch (c) {
            case 's': if (timer.is_running()) timer.pause();
                      else                    timer.start();     break;
            case 'r': timer.reset();                             break;
            case 'f': focus_min = std::max(1,   focus_min - 1);  break;
            case 'F': focus_min = std::min(240, focus_min + 1);  break;
            case 'b': short_min = std::max(1,   short_min - 1);  break;
            case 'B': short_min = std::min(120, short_min + 1);  break;
            case 'q': case '\x1b': restore_terminal(); return 0;
        }
    }
    if (was_running && !timer.is_running() && timer.seconds_left() == 0)
        advance_session();
    was_running = timer.is_running();
    render(timer, session, pomodoro_count, focus_min, short_min, long_min);
    std::this_thread::sleep_for(std::chrono::milliseconds(200));
}

tiny_pomodoro_pro_plus_ultra — countdown thread


countdown_thread:
.loop:
    cmp dword [rel g_stop_thread], 1
    je  .exit
    cmp dword [rel g_state], STATE_RUNNING
    jne .sleep_short              ; paused → poll every 100 ms
    lea rdi, [rel ts_1s]          ; running → sleep exactly 1 second
    xor rsi, rsi
    mov rax, SYS_nanosleep
    syscall
    cmp dword [rel g_state], STATE_RUNNING
    jne .loop                     ; state changed while we slept
    mov eax, [rel g_remaining]
    test eax, eax
    jz  .loop
    dec eax
    mov [rel g_remaining], eax
    jnz .loop
    mov dword [rel g_state], STATE_STOPPED
    jmp .loop

tiny_pomodoro_pro_plus_ultra — itoa, by hand


; edi → decimal string in itoa_buf.  There is no printf here.
itoa:
    mov  eax, edi
    lea  rbx, [rel itoa_buf + 14]
    mov  byte [rbx], 0            ; NUL, then fill backwards
    dec  rbx
.div_loop:
    xor  edx, edx
    mov  ecx, 10
    div  ecx                      ; eax = quotient, edx = remainder
    add  dl, '0'                  ; remainder → ASCII digit
    mov  [rbx], dl
    dec  rbx
    test eax, eax
    jnz  .div_loop
    inc  rbx                      ; rbx → first digit
    ret

tiny_pomodoro_pro_plus_ultra — raw terminal


setup_raw_terminal:
    mov rax, SYS_ioctl
    xor rdi, rdi                  ; stdin
    mov rsi, TCGETS
    lea rdx, [rel orig_termios]
    syscall                       ; read the current settings

    lea rdi, [rel raw_termios]
    mov eax, [rdi + 12]           ; c_lflag lives at offset 12
    and eax, ~(ICANON | ECHO)     ; no line buffering, no echo
    mov [rdi + 12], eax

    mov rax, SYS_ioctl
    xor rdi, rdi
    mov rsi, TCSETS
    lea rdx, [rel raw_termios]
    syscall                       ; write them back
    ret

The assembler

assembler
  • nasm -Ox — multi-pass optimiser: shortest encoding for every instruction
  • ld -nostdlib -static -s — no libc, no runtime, no symbols

NASM's single forward pass conservatively picks the longest encoding when a jump target is still unknown. -Ox iterates to a fixed point — a jmp short is 2 bytes, a jmp near is 5.
-nostdlib is the big one here: the C runtime alone is ~10 KB before you write a line.

The page-alignment floor

  • Linux loads ELF segments via mmap()
  • Each LOAD segment gets its own 4 KiB page
  • Three segments (R, R+X, R) → three pages
  • The 15 flags shrink code, not file

Below ~8 KB you're not optimizing — you're hand-crafting ELF.
See Brian Raiter's Teensy ELF — a working executable in 45 bytes.
